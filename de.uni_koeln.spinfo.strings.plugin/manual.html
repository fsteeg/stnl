Suffix Trees for Natural Language GUI

Requires:

  • Java 1.5.0 or newer
  • Eclipse 3.2 (good chances are Eclipse 3.1.x still works, but it's no longer tested)
  • GraphViz (Dot) for graphics (else plain text)

Usage

  • Input (large): Input text, consisting of one or many words or sentences.
  • Checkbox "word-based": Toggle word-based (for syntactic strucures) and char-based (for morphological structures)
  • Checkbox "reverse": Reverse input text, to see paradigm in the beginning of sentences or words.
  • Button "Suffix Tree": Constructs and visualizes the suffix tree for the input text. Right-click the image to save it to a file.
  • Label "Length": Indicates the length (see Gusfield 1997:286) of the tree.
  • Button "Dot": Save displayed graphic as a dot-file.
  • Input (smaller): Search pattern
  • Input (smallest): Value for k when matching with k-mismatch.
  • Button "k-mismatch": Matching with k-mismatch (see Gusfield 1997:200).
  • Button "wildcards": Matching with wildcards (see Gusfield 1997:199).
  • Button "Reset": Reset directory for "dot" and reset default values for input fields.
  • Button "File": Select a text file to build a suffix tree from (uses the linear-time algorithm), this is always word-based and displayed as plain text.
  • Button "Help": Opens this page.
