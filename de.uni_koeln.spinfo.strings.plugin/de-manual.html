Suffixbäume und Ähnlichkeitsmaße

API und Code

Vorraussetzungen: Java 1.5.0 oder neuer

Über die API können mit linear wachsender Laufzeit (basierend auf dem Algorithmus von Anderssen et al. und UkkoneneSuffixTree der BioJava API) wortbasierte Suffixbäume konstruiert und als Dot exportiert werden werden. Für diese ist ein LCA-Retrieval in konstanter Zeit implementiert (basierend auf einer Implementation von A.G. McDowell). Es bestehen Implementationen von k-mismatch (siehe Gusfield 1997:200) und wildcards (siehe Gusfield 1997:199) Matching sowie eine experimentelle Implementation der Ermittlung der LCE über den LCA (siehe Gusfield 1997:196f.). Darüberhinaus können kleine Suffixbäume naiv in quadratischer Laufzeit sowohl buchstaben- sowie wortbasiert konstruiert und als Dot-Format exportiert werden.

Nutzung

  • Die Quelltexte liegen im Ordner "src" (in das Eclipse-Projekt kopieren, Rechtsklick -> build path -> use as source folder). Javadoc liegt bei, im Ordner "javadoc".

Oberfläche

Vorraussetzungen: Eclipse 3.1.x oder neuer, Java 1.5.0 oder neuer, GraphViz (Dot) für graphische Darstellung (sonst nur Text)

Installation: Ordner "de.uni_koeln.spinfo.strings.plugin" in eclipse/plugins kopieren, Eclipse neu starten, Window -> Show View -> Other -> Stringverarbeitung -> Suffixbäume und Ähnlichkeitsmaße. Beim ersten Öffnen muss der Pfad zu Dot gesetzt werden (zum neu setzen Knopf "Reset", s.u.) Unter Windows ist der Default-Pfad "C:\Programme\ATT\GraphViz\bin", am Mac innerhalb des Programms GraphViz "Content/MacOS". Der View kann mit Control-m maximiert werden.

Die Oberfläche ermöglicht eine flexible Erstellung und Darstellung von Suffixbäumen aus unmittelbar eingegebenem oder in einer Datei gespeicherten Text. Darüber hinaus kann mit zwei Matching-Algorithmen auf dem eingegebenen Text experimentiert werden.

Nutzung

  • Eingabefeld (gross): Eingabetext, bestehend aus einem oder mehreren Wörtern oder Sätzen (z.B. "Er hat gestern sehr gefroren. Sie hat heute sehr gelacht. Es hat letztens sehr geregnet.").
  • Checkbox "wortbasiert": Baum soll Wörter statt Buchstaben enthalten, für Darstellung syntaktischer Strukturen.
  • Checkbox "umkehren": Eingabetext soll umgedreht werden, Für Paradigmen am Anfang.
  • Knopf "Suffixbaum": Konstruiert den Suffixbaum für den eingegebenen Text und stellt ihn dar. Per Rechtsklick auf das Bild kann es abgespeichert werden.
  • Anzeige "Länge": Zeigt die Länge des Baumes an (siehe Gusfield 1997:286).
  • Knopf "Dot": Dargestellten Baum oder Matching-Ergebnis als Dot-Datei abspeichern.
  • Eingabefeld (mittel): Zu suchender Text (Suchmuster, Pattern) Für wortbasiertes Matching mit wildcards etwa: "Es hat letztens sehr *" oder beim k-mismatch "Es hat letztens sehr geregnet.".
  • Eingabefeld (klein): Wert für k beim k-mismatch.
  • Knopf "k-mismatch": Matching mit k-mismatch (siehe Gusfield 1997:200).
  • Knopf "wildcards": Matching mit wildcards (siehe Gusfield 1997:199).
  • Knopf "Reset": Verzeichnis für Dot neu auswählen, Default-Werte für Textfelder wiederherstellen.
  • Knopf "Datei": Auswahl einer Textdatei zum Aufbau eines großen Suffixbaumes (verwendet den linearen Algorithmus), ist immer wortbasiert und wird als Dot-Text-Datei gespeichert und angezeigt.
  • Knopf "Hilfe": Öffnet diese Seite.
